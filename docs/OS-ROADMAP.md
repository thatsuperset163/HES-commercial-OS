# HES peer operating systems (Work)

Equal peers. Each owns one job. Personal is parked for now.

| Order | OS | Owns | Route |
|------|-----|------|--------|
| 1 | **Jobs** | Schedule → run → done | `/work/jobs` |
| 2 | **Money** | Unbilled + unpaid → cash | `/work/money` |
| 3 | **Inbox / Leads** | Same-day inbound response | `/work/leads` |
| 4 | **Reputation** | Reviews / referrals after jobs | `/work/reputation` |
| — | **Sales** | Commercial pipeline & outreach | `/work/sales/` (live) |
| — | **Work home** | Launcher + merged next actions | `/work` |

Sales stays a peer alongside these; we’re filling the missing Work OSes in the order above.

## Rules for every OS

1. Real entities + statuses (not chore checklists)  
2. “Do this next” when it helps  
3. One-tap actions that change state  
4. Logging is a side effect of doing the work
